## Source-Of-Truth Notes

- R102 is the current final live preflight composition surface.
- R103 records approval intent; it does not authorize execution.
- R104 records dry-run evidence; it does not make the system live-ready.
- R105 defines protocol prerequisites; it does not place orders.
- R106 reports activation readiness for a future phase; it does not enable execution.
- R107 is design only and creates no execution authority.
- R108 is a cockpit and intent ledger only; it cannot place orders or override R106.
- R109 hardens R108's cockpit and sacred button semantics; it remains intent-only and non-executing.
- R110 is a burn-down planner over R102-R109; it writes a report ledger and remains non-executing.
- R111 maps R110 burn-down groups into prerequisite checks and writes a prerequisite-clearing ledger; it is not order placement and does not create execution authority.
- R112 records operator evidence for R111 prerequisite needs and reports whether the evidence is ready for R113 recheck; it is not live readiness and does not create execution authority.
- R113 rechecks R111 groups using R112 evidence and writes a recheck ledger; it does not replace R106, change R109, or create execution authority.
- R114 generates exact R112 evidence-recording commands and recheck commands for the active tuple; it does not record evidence automatically, wire evidence to execution, or create execution authority.
