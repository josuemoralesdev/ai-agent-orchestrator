## Source-Of-Truth Notes

- R102 is the current final live preflight composition surface.
- R103 records approval intent; it does not authorize execution.
- R104 records dry-run evidence; it does not make the system live-ready.
- R105 defines protocol prerequisites; it does not place orders.
- R106 reports activation readiness for a future phase; it does not enable execution.
- R107 is design only and creates no execution authority.
- R108 is a cockpit and intent ledger only; it cannot place orders or override R106.
- R109 hardens R108's cockpit and sacred button semantics; it remains intent-only and non-executing.
- R110 is a burn-down planner over R102-R109; it writes a report ledger and remains non-executing.
- R111 maps R110 burn-down groups into prerequisite checks and writes a prerequisite-clearing ledger; it is not order placement and does not create execution authority.
- R112 records operator evidence for R111 prerequisite needs and reports whether the evidence is ready for R113 recheck; it is not live readiness and does not create execution authority.
- R113 rechecks R111 groups using R112 evidence and writes a recheck ledger; it does not replace R106, change R109, or create execution authority.
- R114 generates exact R112 evidence-recording commands and recheck commands for the active tuple; it does not record evidence automatically, wire evidence to execution, or create execution authority.
- R115 wraps R114 output into an ordered operator runbook and command pack; it does not execute commands automatically, wire evidence to execution, or create execution authority.
- R116 assists selected R115 evidence groups and can call R112 evidence recording only after exact evidence-only confirmation; it does not wire evidence to execution or create execution authority.
- R117 rechecks post-evidence state across R112/R116/R113/R111/R110/R106/R109 and writes a diagnostic ledger only; it does not replace R106 or create execution authority.
- R118 performs the final non-executing activation-gate review across R102/R104/R105/R106/R109/R112/R113/R116/R117 and writes a diagnostic ledger only; `READY_TO_REQUEST_FIRST_LIVE_AUTHORIZATION` means a later phase may request explicit authorization, not that live execution is authorized.
- R119 converts R118 blockers into clearing lanes and command packs only; it cannot request authorization, cannot authorize execution, cannot place orders, and writes a diagnostic workbench ledger only.
- R120 consumes R119/R118 and emits a targeted clearing pack or authorization-prep readiness signal only; it cannot request authorization while R118 is blocked, cannot authorize execution, cannot place orders, and writes a diagnostic targeted-clearing ledger only.
- R122 introduces autonomous lane-control intent only; it consumes live eligibility recommendations and keeps `tiny_live` blocked behind existing global gates.
- R123 routes fresh local signal/candidate records into R122 lane-control decisions only; it cannot create orders, authorize execution, or bypass global gates.
