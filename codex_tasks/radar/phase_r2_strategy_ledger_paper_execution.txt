# Radar Phase R2 — Strategy Ledger + Paper Execution

## Goal

Add a deterministic paper execution layer to Hammer Radar so accepted tradable signals can become inspectable paper positions with lifecycle events.

This phase turns Radar from a scanner/evaluator into a strategy-measurement system.

## Current Context

Hammer Radar already:
- detects BTCUSDT hammer/shooting-star signals
- enriches signals with bias, trend, EMA context
- gates tradable candidates
- persists signals to logs/hammer_radar/signals.ndjson
- persists next-candle outcomes to logs/hammer_radar/outcomes.ndjson
- runs inside Docker from Phase R1

## Non-Goals

Do not:
- add Binance API keys
- add live trading
- place real orders
- expose public HTTP APIs
- add dashboards
- replace NDJSON with SQLite
- remove current signal/outcome evaluator
- remove Docker or systemd compatibility

## Execution Rules

For this phase, keep it deterministic and simple:

- Only create paper positions for signal.tradable == True
- Default entry mode: fib_618
- Default paper size: 100 USD
- Stop price: signal.invalidation
- Entry price: signal.fib_618
- Position opens immediately when a tradable signal is accepted
- Do not simulate Binance fills yet
- Do not add leverage yet

## Lifecycle Rules

On each scanner loop:
- after signal creation and tradable decision
- if signal.tradable is true
- create one paper position unless it already exists

Then evaluate open positions against the latest relevant resampled candle:
- long stops if candle low <= stop_price
- short stops if candle high >= stop_price
- otherwise remain open
- if stopped, close position at stop_price
- record pnl_pct and pnl_usd
- append a close event

No take-profit yet. That belongs later.

## Reporting

Print compact operator lines when:
- a paper position opens
- a paper position closes

Example:

PAPER OPEN [13m] BTCUSDT LONG entry=fib_618 price=123.45 size=100.00 stop=120.00 signal=<signal_id>

PAPER CLOSE [13m] BTCUSDT LONG reason=stop exit=120.00 pnl=-2.79% usd=-2.79 signal=<signal_id>
