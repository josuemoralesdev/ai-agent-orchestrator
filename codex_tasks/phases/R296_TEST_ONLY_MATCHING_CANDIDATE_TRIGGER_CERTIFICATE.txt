Purpose:
  Certify that when a test-only matching fresh candidate is injected for an already allowed live-qualified lane, the system records a dry-run simulated trigger lifecycle while still forbidding:
    - live order
    - test order
    - submit
    - final command
    - executable payload
    - secret/signature output
    - env/config/risk mutation

Hard goal:
  Add a test-only matching-candidate trigger certificate for exact-lane dry-run simulation:
    - matching lane => simulated dry-run trigger certificate
    - non-matching lane => no trigger
    - near-miss/paper/unknown lane => blocked
    - all safety flags remain no-submit/no-order/no-executable-payload

  - Machine auto-triggers only when armed and all gates are open.
  - Operator does not approve each signal.
  - R296 is test-only dry-run certification, not live trading.
