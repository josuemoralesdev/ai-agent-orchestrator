# R278 — Operator Dry-Run Arming Control for Real Candidate Watch

Phase classification:
  Primary: OPERATOR ARMING CONTROL / REAL-CANDIDATE DRY-RUN
  Secondary: AUTONOMOUS WATCH, SAFETY HARDENING, PRE-BINANCE READINESS
  Duplicate risk level: MEDIUM

Why this phase exists:
  R275 created the autonomous armed-lane dry-run gate.
  R276 proved fixture rehearsals can reach AUTO_DRY_RUN_READY safely.
  R277 bound the autonomous dry-run engine to real candidate watcher flow and correctly returned WAIT when no fresh real candidate existed.

  The remaining missing pre-live piece is a safe operator arming control:
    - operator can arm exact LIVE_QUALIFIED lanes in DRY-RUN ONLY mode
    - machine waits for real fresh candidates
    - if a real fresh candidate appears for an armed lane, machine prepares simulated execution immediately
    - no real order is sent
    - no Binance order endpoint is called

Hard goal:
  Add a safe operator control path to arm/disarm autonomous real-candidate dry-run lanes.

  Required behavior:
    - default remains fully OFF
    - arming is explicit
    - arming is dry-run only
    - arming is exact-lane only
    - live-qualified lanes only may be armed
    - near-miss / paper-only / below-55 / Betrayal / inverse lanes cannot be armed
    - after arming, the real candidate watch path can evaluate automatically
    - if no real candidate exists, status remains WAIT
    - if a real fresh LIVE_QUALIFIED candidate exists for armed lane, status becomes AUTO_DRY_RUN_READY
    - final_command_available=false
    - submit_allowed=false
    - real_order_forbidden=true
    - order_placed=false
    - binance_order_endpoint_called=false
    - binance_test_order_endpoint_called=false

Non-negotiable safety:
  - Do NOT place real orders.
  - Do NOT call Binance /fapi/v1/order.
  - Do NOT call Binance test order endpoint.
  - Do NOT submit signed requests.
  - Do NOT create HMAC signatures.
  - Do NOT expose secrets.
  - Do NOT mutate env files.
  - Do NOT enable real live execution.
  - Do NOT disable global kill switch.
  - Do NOT make final command available.
  - Do NOT allow arming all lanes with a wildcard.
  - Do NOT allow 8m short arming.
  - Do NOT allow near-miss or paper-only arming.
  - Do NOT allow Betrayal/inverse lane arming.
  - Do NOT persist dangerous live flags.
  - If uncertain, keep arming rejected and return BLOCKED.

Allowed arming lanes:
  These are current live-qualified lanes from prior R274/R277 evidence:
    - BTCUSDT|44m|long|ladder_close_50_618
    - BTCUSDT|44m|short|ladder_close_50_618
