Critical operator correction:
  The target architecture is NOT per-signal operator review.
  The operator does not approve each trade.
  The operator tunes, arms, disarms, configures risk, and watches the machine.
  When all gates are open, the machine must trade by itself by design.

Correct operator role:
  - operator configures risk contracts
  - operator arms/disarms lanes or global mode
  - operator monitors dashboards/alerts
  - operator kills the system if needed
  - operator does NOT manually approve each signal once autonomous mode is armed

Correct machine role:
  - machine watches live-qualified signals
  - machine validates gates
  - machine opens position automatically when armed and gates are open
  - machine closes via protective stop/take-profit/trailing/exit logic automatically
  - machine records audit packets
  - machine alerts operator for visibility, not permission

Why this phase exists:
  R286 used too much "operator review" wording.
  That is acceptable for pre-live safety, but wrong for final architecture.
  R287 must correct terminology and build the autonomous trigger loop as a dry-run/no-submit execution path:
    - if gates are open and lane is dry-run armed, machine creates a simulated execution lifecycle automatically
    - no operator approval prompt
    - no live order
    - no Binance order endpoint
    - no final live command
    - alert is visibility only

  4. Decision logic:
     If no fresh candidate:
       status = AUTONOMOUS_TRIGGER_WAIT

     If fresh candidate exists but paper-only/near-miss/not approved:
       status = AUTONOMOUS_TRIGGER_BLOCKED
       blockers explain strategy/lane reason

     If fresh approved candidate exists but lane not dry-run armed:
       status = AUTONOMOUS_TRIGGER_BLOCKED
       blockers include exact_lane_not_armed
       next_required_step = ARM_APPROVED_LANE_DRY_RUN_ONLY_IF_OPERATOR_WANTS_MACHINE_ACTIVE
       This is NOT per-signal approval. This is machine arming.

     If fresh approved candidate exists, exact lane is dry-run armed, risk contract valid, Binance readiness green, idempotency clean:
       status = AUTONOMOUS_TRIGGER_DRY_RUN_EXECUTED
       record a simulated lifecycle:
         simulated_open_record
         simulated_protective_orders
         simulated_close_plan
       final command still unavailable.
       no order endpoint.
