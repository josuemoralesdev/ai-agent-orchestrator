  - loss_rate_pct
  - outcome_coverage_pct
  - outcome_freshness_status
  - sample_size_bucket
  - win_rate_quality_bucket
  - outcome_quality_score
  - capture_readiness_score
  - combined_watch_score
  - blocker list

Sample size bucket:
- NONE: known_outcome_count == 0
- TINY: 1-29
- SMALL: 30-99
- MEDIUM: 100-299
- LARGE: 300+

Win rate quality bucket:
- UNKNOWN if no known outcomes
- WEAK if win_rate_pct < 55
- MODERATE if 55 <= win_rate_pct < 65
- STRONG if 65 <= win_rate_pct < 75
- VERY_STRONG if win_rate_pct >= 75

Outcome coverage:
known_outcome_count / paper_outcome_count when paper_outcome_count > 0 else null.

Outcome quality score:
- Must be deterministic.
- Must not imply live readiness.
- Suggested rough formula:
  - base win rate component from win_rate_pct
  - sample size bonus
  - outcome coverage bonus
  - freshness bonus
  - unknown outcome penalty
  - blocker penalty
But keep it simple and documented.

Capture readiness score:
- unique_capture_count / threshold_required_count capped at 1.0
- official lane gets official_candidate context but not artificial score inflation

Combined watch score:
- outcome quality plus capture readiness
- must be used for watchlist only, not promotion

Do not mark any lane live-ready.
Do not promote lanes.
