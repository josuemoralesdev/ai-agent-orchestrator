# R275 — Autonomous Armed Lane Dry-Run

Phase classification:
  Primary: AUTONOMOUS EXECUTION ARCHITECTURE / DRY-RUN
  Secondary: FINAL READINESS, OPERATOR ARMING, SAFETY HARDENING
  Duplicate risk level: MEDIUM

Why this phase exists:
  R274 added a live-qualified fresh candidate watcher. That is useful for visibility, but the final architecture must not depend on the operator reading a notification quickly enough.
  The action window is too narrow.
  Target architecture:
    - operator arms or disarms machine/lane
    - when a fresh LIVE_QUALIFIED signal appears and all gates pass, machine executes automatically
    - notifications are for visibility/audit, not the timing-critical trigger
  R275 must build the autonomous armed-lane dry-run path only.
  No real order is allowed in this phase.

Hard goal:
  Build an autonomous dry-run executor that:
    - remains OFF by default
    - requires explicit global auto-live arming and exact lane arming
    - scans live-qualified fresh candidates
    - if armed and all gates pass, creates an autonomous execution dry-run packet
    - never calls Binance order/test-order endpoints
    - never places real orders
    - produces clear audit records and operator visibility

Non-negotiable safety:
  - Do NOT place real orders.
  - Do NOT call Binance /fapi/v1/order.
  - Do NOT call Binance test order endpoint.
  - Do NOT enable real live execution.
  - Do NOT disable global kill switches.
  - Do NOT expose secrets.
  - Do NOT mutate external env files.
  - Do NOT make real final command available.
  - Do NOT allow near-miss lanes to execute.
  - Do NOT allow Betrayal/inverse lanes to execute.
  - Do NOT create an exception for 8m short.
  - Do NOT allow autonomous execution unless BOTH global and exact-lane arming are true.
  - If uncertain, return WAIT / BLOCKED / DRY_RUN_BLOCKED.

Operator architecture:
  Human controls:
    - global_auto_live_enabled
    - lane_auto_live_enabled per exact lane
    - timeframe/direction/entry_mode lane arming
    - max notional
    - max daily trades
    - daily loss stop
    - kill switch
  Machine controls:
    - candidate freshness
    - strategy evidence
    - exact lane qualification
    - risk contract
    - exchange minimum
    - duplicate/idempotency
    - no prior live submit
    - no daily loss breach
    - order payload construction
    - protective stop/take-profit pairing
    - audit + notification

Implementation scope:
  1. New autonomous arming state model:
     Add a safe config/ledger-backed model, preferably not env-based:
       global_auto_live_enabled: false by default
       auto_execute_mode: "dry_run_only" for R275
       lane_auto_live_enabled: false by default for every lane
       armed_lane_key: null unless explicitly armed
       allowed_lane_keys: exact list only
       max_position_notional_usdt: 80.0
       leverage: 10.0
       max_trades_per_day: 1
       daily_loss_stop_usdt: 5.0
       require_protective_orders: true
       require_strategy_live_qualified: true
       require_no_betrayal: true
       require_not_near_miss: true

     Store this in a safe config file or existing lane_controls structure only if consistent with prior design.
     Default must be fully OFF.

  2. Autonomous dry-run executor:
     Add an operator module/function, e.g.:
       tiny_live_autonomous_armed_dry_run.py

     It should:
       - load auto arming state
       - load live-qualified fresh candidate watcher packet
       - select current fresh LIVE_QUALIFIED candidate only
       - verify exact lane is armed
       - verify global auto live is armed
       - verify strategy evidence:
           win_rate >= 55
           sample_count >= 30
           avg_pnl > 0
       - verify not near-miss
       - verify not paper-only
       - verify not Betrayal/inverse
       - verify risk contract:
           explicit_notional_cap_with_leverage
           max notional <= 80
           leverage == 10
           margin budget ~= 8
       - verify exchange minimum can clear
       - verify no duplicate and no prior live submit
       - build simulated order triplet:
           long: BUY market, SELL stop reduce-only, SELL TP reduce-only
           short: SELL market, BUY stop reduce-only, BUY TP reduce-only
       - record autonomous dry-run packet
       - return:
           status:
             AUTO_DRY_RUN_WAIT
             AUTO_DRY_RUN_BLOCKED
             AUTO_DRY_RUN_READY
           real_order_placed=false
           submit_attempted=false
           binance_order_endpoint_called=false
           binance_test_order_endpoint_called=false

  3. API surface:

       GET /tiny-live/autonomous-armed-dry-run

     Optional write/review endpoint only if already consistent with project patterns:
       POST /tiny-live/autonomous-arming/review
     But do NOT make it enable real live. R275 only dry-run mode.

     Output should include:
       arming_state
       candidate_watch_status
       selected_candidate
       strategy_evidence
       exact_lane_risk_contract
       simulated_order_triplet

  5. Final console integration:
