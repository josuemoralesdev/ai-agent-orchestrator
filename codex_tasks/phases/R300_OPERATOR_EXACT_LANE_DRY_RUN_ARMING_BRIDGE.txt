  Duplicate risk level: HIGH

Why duplicate risk exists:
  Existing capabilities already include:
    - R278 operator dry-run arming control
    - R286 fresh live-qualified trigger watch
    - R287 autonomous trigger loop
    - R288 autonomous trigger scheduler
    - R292/R293 timer health
    - R294 dry-run lane arming rehearsal
    - R295 timer-observed armed-lane wait certificate
    - R298 real-candidate dry-run trigger bridge
    - R299 real-candidate timer observation certificate
    - final console panels
  R300 must reuse these. Do not create a parallel arming system, parallel config, parallel scheduler, parallel candidate watcher, or order path.

Current state:
  - R299 committed and clean.
  - Timer is active and observing real-candidate bridge state.
  - R298/R299 currently certify READY_TO_WAIT because no real matching fresh candidate exists.
  - The installed dry-run timer is active but safe.
  - The autonomous arming state is expected to remain OFF unless the operator manually arms it.
  - No final live submit command is available.
  - No order path is available.

Purpose:
  Add an operator-facing exact-lane dry-run arming bridge that:
    - reads current autonomous dry-run arming state
    - verifies whether the requested exact lane is armed for dry-run
    - prints manual operator commands for arming/disarming without executing them
    - certifies blocked/unarmed state safely by default
    - certifies armed dry-run readiness only after operator manually arms outside Codex
    - keeps final command, submit, executable payload, and order placement unavailable

Hard goal:
  If exact lane is not manually armed:
    - status=OPERATOR_EXACT_LANE_DRY_RUN_ARMING_BRIDGE_NOT_ARMED
    - provide print-only manual arming command
    - no config mutation by Codex
    - no order/submit/payload

  If operator has manually armed the exact lane:
    - status=OPERATOR_EXACT_LANE_DRY_RUN_ARMING_BRIDGE_ARMED_CERTIFIED
    - exact_lane_auto_armed=true
    - allowed_lane_keys contains requested lane only or safely includes requested approved lane
    - dry_run_only=true
    - live_execution_enabled=false
    - allow_live_orders=false
    - global kill switch still enforced for live
    - no order/submit/payload

Operator doctrine:
  - Operator arms/disarms/tunes risk/kills system.
  - Machine auto-triggers only when armed and all gates are open.
  - Operator does not approve each signal.
  - R300 remains dry-run arming bridge only.
  - No live execution is authorized.

Non-negotiable safety:
  - Do NOT place orders.
  - Do NOT call Binance /fapi/v1/order.
  - Do NOT call Binance test order endpoint.
  - Do NOT call Binance leverage/margin mutation endpoints.
  - Do NOT enable live execution.
  - Do NOT set HAMMER_LIVE_EXECUTION_ENABLED=true.
  - Do NOT set HAMMER_ALLOW_LIVE_ORDERS=true.
  - Do NOT disable global kill switch.
  - Do NOT mutate env files.
  - Do NOT mutate live config.
  - Do NOT mutate risk contracts.
  - Do NOT mutate configs/hammer_radar/autonomous_arming_state.json from Codex.
  - Do NOT arm or disarm the system from Codex.
  - Do NOT create executable order payloads.
  - Do NOT make final live submit command available.
  - Do NOT print secrets, signatures, signed URLs, API keys, or env values.
  - Do NOT require per-signal operator approval.
  - Do NOT install/start/stop/restart/enable/disable systemd.
  - Do NOT run sudo.
  - Do NOT write to /etc/systemd/system.
  - Do NOT use test-only fake candidate input.
  - R300 must only observe arming state and print manual commands.

Allowed:
  - Modify repo code/docs/tests.
  - Read existing configs/logs.
  - Read current autonomous dry-run arming state.
  - Read R298/R299 status.
  - Read timer health using existing read-only helper.
  - Read scheduler status.
  - Write repo-local R300 ledger only when explicit record flag is used.
  - Add API/CLI/final-console visibility.
  - Add print-only script.
  - Add tests using fixtures/mocks.
  - Produce manual operator command strings that the operator may choose to run outside Codex.
