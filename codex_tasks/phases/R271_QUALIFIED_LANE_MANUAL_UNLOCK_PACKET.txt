    - exposes a manual-only command packet only when all gates pass and exact manual unlock confirmation is provided

    - never submits from Codex

Scope:
  1. Qualified lane selector:
     - Determine current fresh/proposed candidate lane:
         symbol
         timeframe
         direction
         entry_mode
         signal_id
     - Fetch strategy evidence for that exact lane.
     - Lane qualifies only if:
         symbol == BTCUSDT
         win_rate_pct >= 55.0
         sample_count >= 30 or configured min_sample
         avg_pnl_pct > 0 unless existing policy proves another positive-expectancy measure
         entry_mode is valid
         signal origin is allowed
         betrayal/inverse not involved
     - If no current fresh/proposed candidate, remain blocked:
         no_current_qualified_fresh_candidate

  2. Exact lane risk contract:
     - Require exact match:
         symbol + timeframe + direction + entry_mode
     - If missing and lane is qualified, create/apply a guarded exact lane contract:
         tiny_live_contract_mode: explicit_notional_cap_with_leverage
         max_position_notional_usdt: 80.0
         max_notional_usdt: 80.0
         leverage: 10.0
         margin_budget_usdt: 8.0
         max_loss_usdt: 4.44 unless stricter existing policy applies
         live_execution_enabled: false
         live_authorized: false
     - Config write allowed only for the exact lane.
     - Do not mutate lane_controls in this phase unless existing R263-style arming is explicitly required and remains non-submit.

  3. Lane-specific expected order packet:
     - Long:
         main BUY MARKET
         stop SELL STOP_MARKET REDUCE_ONLY
         take_profit SELL TAKE_PROFIT_MARKET REDUCE_ONLY
     - Short:
         main SELL MARKET
         stop BUY STOP_MARKET REDUCE_ONLY
         take_profit BUY TAKE_PROFIT_MARKET REDUCE_ONLY
     - Quantity must be current exchange-valid quantity and notional <= 80.
     - The expected order wording must not be hardcoded to 8m short.

  4. Manual-only unlock packet:
     - Diagnostic mode must keep:
         final_live_submit_command_packet.available = false
     - Manual unlock mode may expose a command only if:
         exact R271 unlock phrase is supplied
         current candidate lane is qualified
         exact lane risk contract valid
         R262B valid
         R263 controls armed
         R264 dry preview valid
         signed triplet fresh
         idempotency clean
         no prior live submit
         no order placed
         no submit attempted
         no Binance order endpoint called
         no test order endpoint called
         no private/transfer/withdraw endpoints
     - Exposed packet must include:
         do_not_run_from_codex=true
         must_be_run_manually_by_operator=true
         submit_allowed_from_codex=false

  5. Final console / API:
     - Surface:
         current qualified lane
         strategy evidence: win_rate/sample/min_sample/avg_pnl
         signal origin and betrayal status
         exact lane risk contract status
         expected orders
         JIT validation
         manual unlock availability
         exact blockers
         safety flags
     - No submit button.
