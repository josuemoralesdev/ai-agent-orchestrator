Current state:
  - R300 committed and clean.
  - R300 proves Codex did not arm or mutate config.
  - Current expected default state before manual operator arming:
      OPERATOR_EXACT_LANE_DRY_RUN_ARMING_BRIDGE_NOT_ARMED

  - No live order path is available.

Purpose:
  Add a post-arm certificate that can be run after the human operator manually arms the exact lane for dry-run. It certifies whether:
    - the requested exact lane is armed for dry-run
    - arming was performed outside Codex
    - dry-run autonomy is ready to wait for a real matching candidate
    - live execution remains disabled
    - final command / submit / order remain unavailable

Hard goal:
  Before manual arming:
    - status=MANUAL_OPERATOR_DRY_RUN_ARMING_POST_ARM_NOT_ARMED
    - no order/submit/payload
    - report manual arming command from R300

  After manual operator arms exact lane outside Codex:
    - status=MANUAL_OPERATOR_DRY_RUN_ARMING_POST_ARM_CERTIFIED
    - exact_lane_auto_armed=true
    - any_lane_auto_armed=true
    - armed_lane_key=requested_lane_key
    - global_auto_live_enabled may be true only in dry_run_only arming context, never live execution
    - live_execution_enabled=false
    - allow_live_orders=false
    - final_command_available=false
    - submit_allowed=false
    - real_order_forbidden=true
    - order_payload_created=false
    - order_placed=false

Operator doctrine:
  - Operator arms/disarms/tunes risk/kills system.
  - Machine auto-triggers only when armed and all gates are open.
  - Operator does not approve each signal.
  - R301 remains dry-run post-arm certification only.
  - No live execution is authorized.
