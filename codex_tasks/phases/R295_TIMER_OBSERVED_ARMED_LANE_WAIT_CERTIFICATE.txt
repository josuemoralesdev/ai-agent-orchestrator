Why duplicate risk exists:
  Existing capabilities already include:
    - autonomous dry-run arming state
    - R294 dry-run lane arming rehearsal
    - autonomous trigger loop
    - autonomous trigger scheduler
    - timer health
    - final console panels
  R295 must reuse these. Do not build a second arming system, second scheduler, second candidate watcher, or second timer-health checker.

Current state:
  - R294 committed and clean.
  - Dry-run systemd timer is active and healthy.
  - R294 dry-run lane arming rehearsal accepts:
      BTCUSDT|44m|long|ladder_close_50_618
      BTCUSDT|44m|short|ladder_close_50_618
      BTCUSDT|55m|long|ladder_close_50_618
  - R294 records READY_TO_WAIT when no matching fresh candidate exists.

  - No live orders, no executable payloads, no final submit command.

Purpose:
  Certify that the running timer can observe an armed dry-run lane state across scheduler ticks and that it remains safe while waiting for a matching fresh live-qualified candidate.

Hard goal:
  Add an audit/certificate layer proving:
    - a selected dry-run lane is armed or expected for observation
    - timer is active
    - recent scheduler ticks exist
    - recent ticks saw WAIT state safely
    - no matching fresh candidate exists yet, or if one exists, it is exact-lane checked
    - no cross-lane borrowing
    - no submit/order/executable payload can happen
    - final console summarizes this as operator-ready wait state

Operator doctrine:
  - Operator arms/disarms/tunes risk/kills system.
  - Machine auto-triggers only when armed and all gates are open.
  - Operator does not approve every individual signal.
  - R295 remains dry-run observation/certification only.
