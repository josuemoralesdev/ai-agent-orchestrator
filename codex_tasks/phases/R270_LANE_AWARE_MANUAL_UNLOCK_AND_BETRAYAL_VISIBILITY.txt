Scope:
  1. Lane-aware unlock selection:
     - Determine current proposed ticket:
         symbol
         timeframe
         direction
         entry_mode if available
         signal_id
     - Manual unlock packet must be generated for this exact lane only.
     - If current ticket is BTCUSDT 4m long, do not use BTCUSDT 8m short phrases/orders.
     - Expected order wording must reflect actual side:
         long -> main BUY MARKET, exits SELL reduce-only
         short -> main SELL MARKET, exits BUY reduce-only
     - Candidate notional must stay <= 80 USDT.
     - Leverage must remain 10x.
     - No final command in diagnostic mode.

  2. Betrayal / signal-origin visibility:
     - Surface in trade ticket and final console:
         signal_origin_family
         betrayal_mode_involved true/false/unknown
         betrayal_inverse_involved true/false/unknown
         promotion_family/status if available
         whether this candidate is standard checklist, betrayal-derived, inverse-derived, or unknown
     - If Betrayal/inverse is involved:
         - block final manual unlock unless an explicit operator acceptance gate exists for betrayal first tiny live
         - explain blocker as betrayal_first_tiny_live_not_explicitly_accepted
     - If origin is unknown:
         - block final manual unlock or mark needs_manual_origin_review
       unless existing project policy already allows unknown origin. Prefer safety.

  3. Final console:
     - Show:
         current proposed ticket lane
         lane-specific expected orders
         signal origin / betrayal status
         R262B/R263/R264 status
         signed triplet freshness
         80 USDT / 10x contract
         exact blockers
         final manual command availability
     - No submit button.

  4. JIT:
     - Diagnostic mode stays safe.
     - Manual unlock mode may expose manual-only command only when:
         current ticket lane matches generated packet
         signal origin is allowed
         no betrayal/inverse uncertainty remains
         all R269 gates pass
         exact unlock phrase is supplied
     - Command must remain:
         do_not_run_from_codex=true
         must_be_run_manually_by_operator=true

  5. Tests:
     Add/update tests proving:
       - 4m long ticket gets BUY main / SELL reduce-only expected order wording.
       - 8m short packet is not used for 4m long ticket.
       - mismatch blocks command availability.
       - betrayal/inverse involvement blocks unless explicitly accepted.
       - unknown origin blocks or is clearly marked according to safest policy.
       - standard allowed origin can proceed to manual-only availability when all gates and exact phrase pass.
       - no order placed.

       - no Binance order endpoint called.
       - secrets_shown=false.
