ROUTER CONCEPT:
A routed signal/candidate must be evaluated as:

lane key:
symbol + timeframe + direction + entry_mode

Route result:
- ROUTED_TO_LANE
- BLOCKED_BY_LANE
- EXPIRED_SIGNAL
- NO_MATCHING_LANE
- ROUTER_NO_CANDIDATE
- ROUTER_ERROR

Freshness:
Use the lane's freshness_seconds.
A candidate is fresh only if now - candidate_timestamp <= freshness_seconds.

Modes:
- disabled: never route
- paper: route as paper-only candidate
- armed_dry_run: route as dry-run candidate only
- tiny_live: still must be blocked unless global gates are ready; R123 does not execute
