GOAL:
Create the non-executing autonomous lane control loop scaffold.

This phase converts the architecture from manual per-signal approval toward the correct operating model:

Operator arms/disarms lanes.
System autonomously watches fresh signals.
System decides entry eligibility, size/risk intent, paper/dry-run/tiny-live intent, and exit policy intent.
System records decisions.
No real orders are placed in R127.

The operator controls:
- global kill switch
- per-lane switch/mode
- per-timeframe/strategy authorization

The system controls:
- fresh signal detection
- lane matching
- entry decision
- size/risk intent
- stop/take-profit policy intent
- cooldown enforcement
- max daily trades/loss enforcement
- paper/dry-run/tiny-live intent selection
- decision ledgering

CORE LOOP MODEL:
For each lane:
1. Read lane mode.
2. Read fresh routed candidates.
3. Match candidate to lane.
4. Check freshness.
5. Check lane mode.
6. Check lane limits:
   - max_daily_trades
   - max_daily_loss_pct
   - cooldown_after_loss_minutes
   - require_protective_orders
7. Choose autonomous decision:
   - IGNORE
   - PAPER_OBSERVE
   - PAPER_ENTRY_INTENT
   - ARMED_DRY_RUN_INTENT
   - TINY_LIVE_GATE_REVIEW
   - BLOCKED
8. Determine non-executing strategy intent:
   - entry_reference if available
   - stop_reference if available
   - take_profit_reference if available
   - size_policy: from risk limits, not direct live quantity
   - exit_policy: protective orders required / preview only
9. Record decision to append-only ledger.
10. Return compact operator status.

Default behavior:
- preview only
- no decision ledger write
- compact status

Supported args:
- --record-decision
- --lane-key <lane_key>
- --all-lanes
- --confirm-decision-record "<exact phrase>"

Exact confirmation phrase:
I CONFIRM AUTONOMY DECISION RECORDING ONLY; NO ORDER; NO BINANCE CALL.

Behavior:
1. Without --record-decision:
   - status: LANE_AUTONOMY_PREVIEW
   - no decision ledger write
   - show decisions that would be recorded
   - safety fields false

2. With --record-decision but missing/wrong confirmation:
   - status: LANE_AUTONOMY_REJECTED
   - no decision ledger write
   - rejection_reason: missing or invalid decision-record confirmation

3. With --record-decision and exact confirmation:
   - status: LANE_AUTONOMY_RECORDED or LANE_AUTONOMY_PARTIAL
   - append decision records only
   - never place orders
   - never create order payloads
   - never call Binance
   - never mutate env
   - return recorded decision_ids
