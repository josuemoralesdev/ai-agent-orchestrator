Current R296 issue:
  R296 mostly passes, but verification showed that:
    GET /tiny-live/test-only-matching-candidate-trigger-certificate/status
  returns a simulated matching certificate by default, with:
    status=TEST_ONLY_MATCHING_CANDIDATE_TRIGGER_CERTIFIED
    simulation_flag_required=false
    test_only_matching_requested=true
    record_test_only_matching_candidate_trigger_certificate_requested=true
    recorded_at_utc present

Why this must be fixed:
  The API status endpoint must be safe/read-only by default.
  A plain GET status must not inject a simulated candidate, must not record a certificate, and must not imply a matching trigger was requested.

Non-negotiable safety:
  - Do NOT place orders.
  - Do NOT call Binance order/test-order/leverage/margin endpoints.
  - Do NOT enable live execution.
  - Do NOT mutate env/config/risk contracts.
  - Do NOT make final command available.
  - Do NOT create executable payloads.
  - Do NOT run sudo/systemctl/install commands.

Required behavior:
  1. Plain GET:
     /tiny-live/test-only-matching-candidate-trigger-certificate/status

     Must return safe default:
       status = TEST_ONLY_MATCHING_CANDIDATE_TRIGGER_BLOCKED
       blockers includes missing_test_only_simulation_flag
       simulation_flag_required = true
       test_only_matching_requested = false
       test_only_nonmatching_requested = false
       simulated_fresh_candidate_injected = false
       simulated_trigger_recorded = false
       record_test_only_matching_candidate_trigger_certificate_requested = false
       test_only_matching_candidate_trigger_certificate_recorded = false
       recorded_at_utc absent/null
       final_command_available = false
       submit_allowed = false
       real_order_forbidden = true
       executable_payload_created = false
       order_payload_created = false
       order_placed = false

  2. Explicit API query param may simulate but must not record:
     /tiny-live/test-only-matching-candidate-trigger-certificate/status?simulate_matching_for_tests_only=true

     Allowed:
       returns TEST_ONLY_MATCHING_CANDIDATE_TRIGGER_CERTIFIED
       test_only_matching_requested=true
       simulated_trigger_recorded=true
       final_command_available=false
       submit_allowed=false
       real_order_forbidden=true

     Not allowed:
       no ledger write
       no record flag true
       no recorded_at_utc
       no live/order/executable payload

  3. CLI behavior from R296 must remain unchanged:
     explicit CLI record flag may record repo-local R296 ledger.
     CLI matching simulation should still certify.
     CLI nonmatching simulation should still no-trigger.
