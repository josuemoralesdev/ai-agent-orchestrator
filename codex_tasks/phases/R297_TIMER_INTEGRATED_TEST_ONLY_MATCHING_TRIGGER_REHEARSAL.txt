Purpose:
  Add a timer/scheduler-integrated rehearsal packet proving that:
    - the system can run a bounded scheduler-style dry-run rehearsal using a test-only matching candidate
    - the rehearsal uses the same timer/scheduler/final-console visibility chain

    - the matching branch records simulated lifecycle only
    - no live order, no submit, no executable payload, no Binance order/test-order call occurs
