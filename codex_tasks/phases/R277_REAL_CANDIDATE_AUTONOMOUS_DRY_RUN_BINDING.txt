# R277 — Real Candidate Autonomous Dry-Run Binding

Phase classification:
  Primary: AUTONOMOUS EXECUTION BINDING / REAL-CANDIDATE DRY-RUN
  Secondary: SAFETY HARDENING, FINAL PRE-LIVE READINESS, WATCHER INTEGRATION
  Duplicate risk level: MEDIUM

Why this phase exists:
  R275 created the autonomous armed-lane dry-run gate.
  R276 proved the autonomous dry-run path behaves correctly under fixture rehearsals:
    - 44m long fixture => AUTO_DRY_RUN_READY
    - 44m short fixture => AUTO_DRY_RUN_READY
    - 55m long fixture => AUTO_DRY_RUN_READY
    - 8m short near-miss fixture => blocked
    - no submit, no order, no Binance order endpoint

  R277 must now bind the autonomous dry-run path to real fresh candidates from the live-qualified watcher.
  This is still dry-run only.
  No real order is allowed.

Hard goal:
  When a real fresh LIVE_QUALIFIED candidate appears from the watcher and the exact lane is armed in dry-run mode, the system should produce:
    AUTO_DRY_RUN_READY
    selected real candidate
    exact strategy evidence
    exact risk contract validation
    simulated order triplet built from the real signal
    final_command_available=false
    submit_allowed=false
    real_order_forbidden=true
    order_placed=false
    binance_order_endpoint_called=false

  When no real fresh candidate exists:
    AUTO_DRY_RUN_WAIT

  When a real candidate exists but lane/global arming is OFF:
    AUTO_DRY_RUN_BLOCKED or WAIT with explicit arming blocker

Non-negotiable safety:
  - Do NOT place real orders.
  - Do NOT call Binance /fapi/v1/order.
  - Do NOT call Binance test order endpoint.
  - Do NOT submit signed requests.
  - Do NOT enable real live execution.
  - Do NOT disable global kill switch.
  - Do NOT expose secrets.
  - Do NOT mutate env files.
  - Do NOT make final command available.
  - Do NOT allow fixture mode to leak into real candidate mode.
  - Do NOT use rehearsal candidate as real market signal.
  - Do NOT allow near-miss lanes to bind as real ready.
  - Do NOT allow 8m short to bind as live-ready.
  - Do NOT allow Betrayal/inverse lanes to bind as first live.
  - If uncertain, return AUTO_DRY_RUN_WAIT or AUTO_DRY_RUN_BLOCKED.

Implementation scope:
  1. Real candidate binding:
     Extend the autonomous dry-run module so non-rehearsal mode uses only the real live-qualified fresh candidate watcher output.

     Required distinctions:
       rehearsal_mode=false
       fixture_candidate=false
       real_market_signal=true only when selected candidate comes from live watcher
       real_candidate_source="qualified_candidate_watch"
       source_signal_id=<real signal id>
       source_lane_key=<real lane key>
       source_age_minutes=<real age>

     The module must reject:
       fixture_candidate=true in real mode
       real_market_signal=false in real mode
       selected candidate missing source signal id
       selected candidate not in LIVE_QUALIFIED
       selected candidate from NEAR_MISS_INCUBATOR
       selected candidate from PAPER_ONLY
       selected candidate from betrayal/inverse family

  2. Arming behavior:
     Default config stays OFF:
       global_auto_live_enabled=false
       lane_auto_live_enabled=false
       auto_execute_mode=dry_run_only

     Add or verify a safe diagnostic path to evaluate “what would happen if this exact real lane were armed in dry-run.”
     This must be explicit and diagnostic-only, not a real live arming mutation unless existing patterns already support safe dry-run arming.

     Preferred CLI option:
       tiny-live-autonomous-armed-dry-run
         --real-candidate-dry-run-bind
         --dry-run-arm-real-candidate-lane
         --record-autonomous-dry-run
         --operator-id local_operator
         --reason "R277 real candidate dry-run binding only; no submit."

     If no fresh real candidate exists, this should return WAIT, not fake one.

  3. Simulated order triplet from real signal:

         dry_run_only=true
