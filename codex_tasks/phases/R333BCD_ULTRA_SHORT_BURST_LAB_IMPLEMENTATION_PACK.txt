  This is not live trading.
  This is not Tiny Live.
  This is not standard Strategy Lab promotion.
  This is not observed expansion.
  This is an isolated paper-only burst lab.

Purpose:
  Build a combined read-only implementation pack that:
    1. Reads existing local signal/evidence surfaces for 4m and 8m candidates.
    2. Produces burst backtest/replay candidate rows using available local data.
    3. Marks rows as sequence_known=false / sequence_unknown when only candle-level data is available.
    4. Computes gross ROE and estimated net ROE fields only when enough fields are available.
    5. Emits no synthetic performance.
    6. Renders a terminal-only visual diagnostic panel.
    7. Produces a preview-only burst risk contract recommendation object.
    8. Keeps live permission false everywhere.

event_type:
  R333B_ULTRA_SHORT_BURST_BACKTEST_ADAPTER

Backtest adapter behavior:
  - Build paper-only burst replay rows from available local signal/evidence surfaces.
  - Candidate timeframes: 4m and 8m.
  - Candidate sides: long and short if source evidence exists.
  - Candidate leverage: 22x, 44x, 88x, 150x.
  - Checkpoints: 22s, 44s, 66s, 88s, 132s, 176s.
  - If exact signal detection timestamp is missing, set detection_timestamp_status=MISSING_OR_APPROXIMATE.
  - If second/tick price path is missing, set sequence_known=false and sequence_status=SEQUENCE_UNKNOWN_CANDLE_ONLY_OR_NO_INTRABAR_PATH.
  - If price path is unavailable, do not invent checkpoint prices.
  - If only paper evidence summary exists, emit SOURCE_SUMMARY_ONLY rows with no fake burst result.
  - Compute formula previews where possible:
      price_move_pct_for_gross_roe = gross_roe_pct / leverage
      fee_drag_roe = fee_pct_round_trip * leverage
      estimated_slippage_roe = slippage_pct_round_trip * leverage
      net_roe = gross_roe - fee_drag_roe - estimated_slippage_roe
  - Mark any formula-only row as MODEL_PREVIEW_NOT_TRADE_RESULT.
  - Never mark MODEL_PREVIEW_NOT_TRADE_RESULT as live-ready.
  - Never mark sequence_unknown rows as live-ready.

Backtest row schema:
  Every burst backtest row must include:
    - burst_row_id
    - strategy_family=ULTRA_SHORT_LEVERAGE_BURST
    - timeframe
    - side
    - lane_key

    - fee_drag_roe

    - gross_roe_target_pct

    - net_roe_formula
    - sequence_known
    - sequence_status
    - result_status:
        SOURCE_SUMMARY_ONLY
        MODEL_PREVIEW_NOT_TRADE_RESULT
