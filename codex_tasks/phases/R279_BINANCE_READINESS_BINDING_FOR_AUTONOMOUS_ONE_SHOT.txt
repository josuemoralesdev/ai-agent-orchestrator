  /home/josue/workspace/kernel/ai-agent-orchestrator-main

Phase classification:
  Primary: BINANCE READINESS BINDING / PRE-LIVE GATE
  Secondary: AUTONOMOUS ONE-SHOT READINESS, READ-ONLY EXCHANGE VALIDATION, SAFETY HARDENING
  Duplicate risk level: MEDIUM

Why this phase exists:
  R275 created autonomous armed-lane dry-run.
  R276 proved fixture rehearsal can reach AUTO_DRY_RUN_READY.
  R277 bound autonomous dry-run to real candidate watcher flow.
  R278 added operator dry-run arming controls and proved:
    - default OFF
    - exact live-qualified lanes can arm in dry-run mode
    - 8m short / near-miss cannot arm
    - disarm returns OFF
    - no Binance order endpoint is called

  R279 must now bind read-only Binance/exchange readiness to the autonomous one-shot path.
  This is still NOT live execution.
  This is not a real order phase.
  This phase prepares the one-shot gate by proving the system can verify exchange facts safely.

Hard goal:
  Add a Binance readiness binding packet for autonomous one-shot that verifies:
    - Binance credentials are configured without exposing secrets
    - account/balance can be read only if explicit confirmation is given
    - exchange info / precision / min notional can be read only if explicit confirmation is given
    - mark price can be read only if explicit confirmation is given
    - BTCUSDT filters are compatible with configured 80 USDT notional cap
    - leverage expectation is 10x, but do not change leverage in this phase
    - margin expectation is isolated, but do not change margin mode in this phase
    - no open conflicting live position if read-only account inspection supports it
    - no order/test-order/private trading endpoint is called
    - autonomous one-shot remains blocked until Binance readiness is green

Non-negotiable safety:
  - Do NOT place real orders.
  - Do NOT call Binance /fapi/v1/order.
  - Do NOT call Binance test order endpoint.
  - Do NOT submit signed order requests.
  - Do NOT create HMAC signatures for trading endpoints.
  - Do NOT change leverage.
  - Do NOT change margin mode.
  - Do NOT cancel orders.
  - Do NOT transfer or withdraw.
  - Do NOT mutate env files.
  - Do NOT expose secrets.
  - Do NOT enable real live execution.
  - Do NOT disable global kill switch.
  - Do NOT make final command available.
  - Do NOT arm real live.
  - Do NOT treat read-only readiness as permission to execute.
  - If uncertain, return BINANCE_READINESS_BLOCKED.

Allowed network behavior:
  Only read-only Binance endpoints may be used, and only behind explicit confirmation phrases.
  If existing project code already has a read-only precision/mark-price gate, extend/reuse it.
  Do not invent unsafe direct calls if safe wrappers already exist.
