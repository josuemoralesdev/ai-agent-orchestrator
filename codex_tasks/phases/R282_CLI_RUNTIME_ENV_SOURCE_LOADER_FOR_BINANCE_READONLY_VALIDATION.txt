Why this phase exists:
  R281 discovered that Binance read-only env already exists and is wired for the systemd/API runtime:
    - /home/josue/.config/hammer-radar/binance-readonly.env
    - BINANCE_API_KEY
    - BINANCE_API_SECRET
    - BINANCE_CONNECTOR_MODE
    - BINANCE_LIVE_TRADING_ENABLED

  R281 proved:
    - API/final console sees ACCOUNT_READ_ENV_READY
    - CLI shell does not see those env vars by default
    - no secrets were printed
    - no env files were mutated
    - no order endpoints were called

  R282 must let CLI validation safely load the same already-existing systemd EnvironmentFile so private account/position read-only validation can run from CLI too.

Hard goal:
  Add a safe CLI/runtime env source loader that loads existing discovered read-only env files into process memory only for validation commands, redacts values, and then runs R280 account/position read-only checks.

Important:
  Do NOT create new env files.
  Do NOT modify env files.
  Do NOT commit env files.
  Do NOT print secret values.
  Do NOT require manual secret entry.

Non-negotiable safety:
  - Do NOT print API key values.
  - Do NOT print API secret values.
  - Do NOT print full signed URLs.
  - Do NOT write env files.
  - Do NOT mutate env files.
  - Do NOT commit env files.
  - Do NOT place real orders.
  - Do NOT call Binance order endpoint.
  - Do NOT call Binance test order endpoint.
  - Do NOT change leverage.
  - Do NOT change margin mode.
  - Do NOT cancel orders.
  - Do NOT transfer or withdraw.
  - Do NOT enable live execution.
  - Do NOT disable global kill switch.
  - Do NOT make final command available.
  - Do NOT weaken confirmation phrases.
  - Do NOT use non-allowlisted endpoints.
  - Do NOT automatically load arbitrary env files outside discovered safe paths.
  - If path is not allowlisted/discovered from systemd, block.

Allowed env source:
  Discovered env file:
    /home/josue/.config/hammer-radar/binance-readonly.env

  It may be loaded into CLI process memory only when:
    - explicit CLI flag requests runtime env file loading
    - file path matches discovered/allowlisted path
    - no values are printed
    - no env file is modified
    - loaded vars are limited to expected Binance read-only env names
    - connector mode resolves to read_only
    - live trading flag resolves false
