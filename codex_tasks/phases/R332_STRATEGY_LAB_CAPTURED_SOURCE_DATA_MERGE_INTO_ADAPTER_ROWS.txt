D. Merge logic:
  Match R331 source-data rows to R328 adapter rows by:
    1. source_row_id == adapter row_id when exact match exists
    2. lane_key + adapter/capture family + variant/capture name mapping when source_row_id differs
    3. preserve unmatched rows explicitly

  Mapping examples:
    - R328 adapter_id exits maps to R331 capture_adapter_id exit_variant_comparison
    - R328 adapter_id ma_wma_anchor maps to R331 ma_wma_anchor_enrichment
    - R328 adapter_id review_ready_enrichment maps to R331 review_ready_enrichment
    - R328 adapter_id capture_8m_short maps to R331 short_capture_improvement
    - R328 adapter_id near_miss_13m maps to R331 near_miss_variant_capture
    - R328 adapter_id betrayal_inverse_lab maps to R331 betrayal_inverse_source_chain
    - R328 adapter_id watch_88m maps to R331 watch_88m_durability

  Status rules:
    - Adapter ready + source ready => MERGED_READY
    - Source pending => MERGED_PENDING_SOURCE_DATA
    - Lab-only source or adapter => MERGED_LAB_ONLY
    - Watch-only source or adapter => MERGED_WATCH_ONLY
    - Unmatched adapter row => MERGED_UNMATCHED_ADAPTER_ROW
    - Unmatched source row => MERGED_UNMATCHED_SOURCE_ROW

  Do not invent source data.
  Do not convert pending rows to ready.
  Do not create synthetic performance.
