ITERATION FLOW:
For each iteration:
1. collect generated_at
2. for each lane:
   - fresh router status
   - lane autonomy scheduler preview
   - autonomous paper lane executor integration preview
   - R141 post-clearing recheck
   - R138 burn-down light summary
3. determine if any lane has eligible paper decisions
4. if eligible:
   - attempt paper proof capture through R140 safe clearing executor OR directly through R129 safe record path if R140 delegates to R129
   - use exact safe paper confirmation:
     I CONFIRM AUTONOMOUS PAPER LANE INTEGRATION ONLY; NO REAL ORDER; NO BINANCE CALL.
   - verify evidence captured
   - stop loop with FRESH_CANDIDATE_WATCH_CAPTURED_PAPER_PROOF
5. if none eligible:
   - record iteration summary
   - sleep if more iterations remain
6. stop on:
   - proof captured
   - timeout / max iterations
   - safety violation
   - exception
   - operator invalid confirmation

IMPORTANT PAPER PROOF RULE:
R142 must not create proof directly.
It must use R129/R140 safe proof path.
If R129/R140 says no eligible decision, R142 must not force proof.
