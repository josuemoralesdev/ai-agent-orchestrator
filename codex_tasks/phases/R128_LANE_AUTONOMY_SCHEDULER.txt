Command:

  --log-dir logs/hammer_radar_forward \
  lane-autonomy-scheduler

Default behavior:
- preview only
- no scheduler tick ledger write
- no decision ledger write
- compact status

Supported args:
- --once
- --record-tick
- --record-decisions
- --lane-key <lane_key>
- --all-lanes
- --confirm-scheduler-record "<exact phrase>"

Exact confirmation phrase:
I CONFIRM AUTONOMY SCHEDULER RECORDING ONLY; NO ORDER; NO BINANCE CALL.

Behavior:
1. Default:
   - status: LANE_AUTONOMY_SCHEDULER_PREVIEW
   - no tick ledger write
   - no decision ledger write

2. With --record-tick but missing/wrong confirmation:
   - status: LANE_AUTONOMY_SCHEDULER_REJECTED
   - no tick ledger write
   - no decision ledger write

3. With --record-tick and exact confirmation:
   - status: LANE_AUTONOMY_SCHEDULER_TICK_RECORDED
   - write scheduler tick record only
   - if --record-decisions also provided, call R127 decision recording safely using same safety constraints
   - never place orders
   - never create order payloads
   - never call Binance
   - never mutate env

Output JSON must include:
- status
- generated_at
- once
- record_tick_requested
- record_decisions_requested
- confirmation_valid
- selected_lane_keys
