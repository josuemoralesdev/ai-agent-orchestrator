Why duplicate risk exists:
  Existing capabilities already include:
    - R287 autonomous trigger loop
    - R288 autonomous trigger scheduler
    - R294 dry-run lane arming rehearsal
    - R295 timer-observed armed-lane wait certificate
    - R296 test-only matching candidate trigger certificate
    - R297 timer-integrated test-only matching trigger rehearsal
    - fresh live-qualified candidate watcher
    - one-shot pre-activation gate
    - final console
  R298 must reuse these. Do not build a parallel candidate watcher, scheduler, arming system, live gate, or order path.

Current state:
  - R297 committed and clean.
  - R297 proves a timer/scheduler-style path can handle a test-only matching candidate safely.
  - Installed systemd timer is active and remains real-candidate-only.
  - Plain R296/R297 APIs are safe by default and do not simulate or record.
  - No live order path is available.
  - No final submit command is available.

Purpose:
  Add a real-candidate dry-run bridge that certifies the system can evaluate the actual fresh-candidate watcher output against the approved lane and produce one of:
    - REAL_CANDIDATE_DRY_RUN_TRIGGER_BRIDGE_READY_TO_WAIT
    - REAL_CANDIDATE_DRY_RUN_TRIGGER_BRIDGE_CERTIFIED
    - REAL_CANDIDATE_DRY_RUN_TRIGGER_BRIDGE_BLOCKED

Hard goal:
  If no real fresh matching candidate exists:
    - report READY_TO_WAIT
    - no trigger
    - no order
    - no submit

  If a real fresh candidate exists and matches an approved live-qualified lane:
    - report CERTIFIED
    - produce simulated dry-run lifecycle records based on real candidate fields
    - never produce executable order payload
    - never place or submit an order

  If real candidate exists but is nonmatching/near-miss/paper-only/expired:
    - report BLOCKED or WAIT with clear blockers
    - no trigger

Operator doctrine:
  - Operator arms/disarms/tunes risk/kills system.
  - Machine auto-triggers only when armed and all gates are open.
  - Operator does not approve each signal.
  - R298 remains dry-run certification only.
  - Alerts are visibility/audit, not permission.

Non-negotiable safety:
  - Do NOT place orders.
  - Do NOT call Binance /fapi/v1/order.
  - Do NOT call Binance test order endpoint.
  - Do NOT call Binance leverage/margin mutation endpoints.
  - Do NOT enable live execution.
  - Do NOT set HAMMER_LIVE_EXECUTION_ENABLED=true.
  - Do NOT set HAMMER_ALLOW_LIVE_ORDERS=true.
  - Do NOT disable global kill switch.
  - Do NOT mutate env files.
  - Do NOT mutate live config.
  - Do NOT mutate risk contracts.
  - Do NOT create executable order payloads.
  - Do NOT make final live submit command available.
  - Do NOT print secrets, signatures, signed URLs, API keys, or env values.
  - Do NOT require per-signal operator approval.
  - Do NOT install/start/stop/restart/enable/disable systemd.
  - Do NOT run sudo.
  - Do NOT write to /etc/systemd/system.
  - Do NOT use test-only fake candidate input for R298 certification.
  - R298 must only use real fresh-candidate watcher output.

Allowed:

  - Write repo-local R298 ledger only when explicit record flag is used.
